# Code of Conduct

## Our pledge

We want this repository to be a welcoming, safe, and constructive place for users, contributors, and maintainers.

In the interest of an open and healthy community, we pledge to make participation in this project a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, sex characteristics, gender identity or expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Our standards

Examples of behavior that contributes to a positive environment include:

- being respectful and constructive in reviews, issues, discussions, and pull requests
- assuming good intent while still giving clear technical feedback
- sharing context, evidence, and reproduction steps instead of personal attacks
- accepting responsibility and apologizing when we make mistakes
- focusing on what is best for the project and its users

Examples of unacceptable behavior include:

- harassment, trolling, insults, or discriminatory language
- sexualized language or unwelcome sexual attention
- doxxing or publishing private information without permission
- sustained disruption of issues, reviews, or discussion threads
- bad-faith technical argument intended to intimidate or exclude others

## Enforcement responsibilities

Project maintainers are responsible for clarifying and enforcing this Code of Conduct. They may remove, edit, or reject comments, commits, issues, and other contributions that are not aligned with this Code of Conduct.

## Scope

This Code of Conduct applies within project spaces and when an individual is officially representing the project in public spaces.

## Reporting

If you experience or witness unacceptable behavior, contact the project maintainers privately through GitHub maintainer contact options when available.

If you cannot reach a maintainer privately, open a minimal public issue requesting a private follow-up without posting sensitive details.

For security-sensitive concerns, follow [`SECURITY.md`](SECURITY.md) instead of posting details publicly.

## Consequences

Maintainers may take any action they deem appropriate in response to unacceptable behavior, including warning the offender, removing content, temporarily limiting participation, or banning a contributor from project spaces.
